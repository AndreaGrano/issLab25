Introduction

Progetto conwaygui: realizzazione di una pagina HTML che funga da dispostivo di input-ouput evoluto per l’applicazione ConwayLife25 in Java .

Nel passo precedente è stato eseguito il refactoring del modello del dominio applicativo, con l'introduzione delle classi Cell e Grid.

Requirements

È richiesta l'implementazione di una GUI distribuita per l'applicazione ConwayLife25. Questa deve essere nella forma di una pagina HTML, denominato guipage.html, avente funzione di:

  • dispositivo di input, permettendo all'utente umano di impostare lo stato iniziale delle celle e di gestire l'applicazione con pulstanti di start, stop, reset ed exit;
  • dispositivo di output, permettendo di visualizzare lo stato della griglia, anche su più macchine contemporaneamente.

Oltre alla pagina HTML è richiesta la realizzazione di:

  • iomap.js: un file JavaScript per la creazione e la gestione della rappresentazione delle celle;
  • commsocket.js: un file JavaScript per la gestione della WebSocket lato client.

Requirement analysis

L'utente umano deve poter in ogni momento visualizzare lo stato delle celle della griglia. Prima che il gioco cominci, egli dovrà poter modificare lo stato iniziale delle celle; una volta impostato, avvierà il gioco tramite il bottone start. Una volta avviata la partita, anche più utenti devono poter visualizzare l'evoluzione della griglia contemporaneamente. Durante lo svolgimento della partita deve essere possibile interromperla e/o resettare lo stato della griglia e/o uscire dal gioco.

Problem analysis

La presenza concorrente di più utenti impone di trovare un criterio per determinare chi abbia il controllo, al fine di evitare collisioni tra interazioni diverse (e.g. utenti diversi che provassero a modificare contemporaneamente lo stato iniziale di una cella). L'accesso a una partita non è gestito dall'applicativo ConwayLife25, dunque questo è delegato alla GUI e l'utente avente il controllo di una data partita, dunque, sarà quello collegatosi per primo alla pagina HTML.

L'applicazione ConwayLife25 espone un'interfaccia Java, IOutDevices, la quale espone un contratto per un modulo che volesse implementare una GUI; si potrebbe, allora, realiazzare una classe che implementi tale interfaccia. Tuttavia, vi sono dei framework che permettono di implementare pagine HTML che interagiscano con un server senza modificare il software operante su di esso; uno di questi è SpringBoot.

Nei requisiti è stato richiesto l'utilizzo del protocollo WebSocket, necessario al fine di far aggiornare automaticamente la pagina HTML su tutti i client connessi.

Una possibile ottimizzazione della banda richiesta dalla comunicazione potrebbe consistere nell'invio delle sole informazioni riguardanti le celle che in un dato momento cambiano di stato. Tuttavia, un utente che si connettesse a partita già cominciata dovrebbe poter vedere lo stato corrente di tutta la griglia, dunque il server dovrà inviare le informazioni riguardanti l'intera griglia.
